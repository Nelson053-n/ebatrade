# pairsignal

Аналитик-советник по парному спреду криптоперпетуалов + виртуальная биржа (paper).
Читает котировки, считает спред/индикаторы, выдаёт рекомендации на вход, ждёт решения
оператора, ведёт виртуальные сделки. **Phase 1 — без реальных торгов.**

## ⛔ Жёсткие ограничения (не нарушать)
- **Никакого реального размещения ордеров.** Только чтение публичных данных и paper-исполнение.
  Не добавлять `create_order`/приватные эндпоинты биржи без явного запроса в новой фазе.
- **Фьючерсный API MEXC закрыт для розницы** (`code 1002 "Contract not allow place order"`).
  Реальное исполнение в будущем — Bybit/OKX/Bitget/Binance. MEXC использовать только для чтения.
- **Секреты не коммитить.** Ключи — через переменные окружения (`.env`, см. `.env.example`).
- Индикаторы считать **только по закрытым свечам** (no repaint).

## Команды
```bash
pip install -r pairsignal/requirements.txt      # зависимости
python -m pairsignal.main --synthetic --auto     # офлайн-демо без сети
python -m pairsignal.main --synthetic            # с ручным подтверждением входов
python -m pairsignal.main --live                 # реальные котировки (нужна сеть)
uvicorn pairsignal.api:app --reload              # backend под панель
pytest -q                                        # тесты
ruff check pairsignal                            # линт
```

## Архитектура (слои → файлы)
- `config.py` — pydantic-конфиг (пары, пороги, комиссии, сайзинг). Без хардкода.
- `models.py` — dataclass'ы и enum'ы; общий «язык» между слоями.
- `indicators.py` — динамическая β (rolling-OLS), спред, BB, z-score, ширина канала.
- `data_feed.py` — CCXT (чтение OHLCV) + синтетический генератор для офлайн-прогона.
- `strategy.py` — `SignalEngine`: чистый конечный автомат, превращает срез индикаторов в рекомендацию.
- `virtual_exchange.py` — `VirtualExchange`: paper-филлы, комиссии, проскальзывание, P&L, позиции.
- `engine.py` — `Engine`: оркестрация + human-in-the-loop (`approve`/`reject`), сводка.
- `main.py` — CLI-раннер. `api.py` — FastAPI backend.

## Поток
`data → indicators → SignalEngine → Engine(approve/reject) → VirtualExchange → summary`

## Конвенции
- Python 3.12, везде type hints. Конфиг — pydantic v2, не глобальные константы.
- **Вход требует подтверждения оператора** (`Engine.approve()`); **выход и стоп — автоматические**.
- Сайзинг доллар-нейтральный: распределение ног по β, нотационал = `risk_pct`% баланса.
- Логика входа: `−stop_z < z ≤ −entry_z` → лонг A / шорт B; симметрично сверху.

## Подводные камни (узнано на практике)
- В pandas `Series.std`/`.mean` — **методы**, конфликтуют с колонками. Колонки спреда читать
  как `row["std"]`, не `row.std` (см. `Engine.rows_from_df`).
- Не входить, если `|z|` уже за `stop_z` — иначе вход и моментальный стоп (фильтр в `strategy.py`).
- **Синтетический генератор в `data_feed.py` — заглушка.** Его метрики ничего не говорят
  о реальной доходности; для оценки нужен бэктест на исторических данных.
