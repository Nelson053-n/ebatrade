# st6 · Correlation-Gated Pairs — корреляционная стратегия для T-Bank API

Парный mean-reversion с корреляционным гейтом и динамическим отбором пары.
Торгует через **T-Bank Invest API (только песочница, SandboxService)** — реальные
биржевые ордера не отправляются. Спроектировано под контур вашей панели *ebatrade*
(закрытые свечи, no repaint, атомарное исполнение пары, sandbox-ордера).

## Что внутри стратегии

Три корреляционных приёма в одной стратегии:

1. **Отбор пары (ranking).** Из корзины родственных бумаг сканер перебирает все
   пары и оставляет те, где одновременно: высокая корреляция лог-доходностей,
   спред коинтегрирован (тест Дики-Фуллера / ADF) и быстрый half-life возврата.
   Лучшая пара по композитному score идёт в торговлю.
2. **Корреляционный гейт.** Вход разрешён, только пока скользящая корреляция ног
   ≥ `corr_enter`. Если связь распадается (corr < `corr_break`) — позиция
   принудительно закрывается. Это защита от классического слома пар-трейда, когда
   две бумаги перестают ходить вместе и спред больше не возвращается.
3. **Mean-reversion спреда.** `spread = ln(A) − β·ln(B)`, где β — хедж-коэффициент
   по OLS (доллар-нейтральные ноги). Торгуем z-score: вход при `|z| ≥ z_entry`
   против движения, выход при `|z| ≤ z_exit`, стоп при `|z| ≥ z_stop`.

Высокий z → спред дорого → **SHORT спреда** (short A, long B).
Низкий z → спред дёшево → **LONG спреда** (long A, short B).

## Корзина по умолчанию

Ликвидные коррелированные акции MOEX, доступные на T-Bank: нефтегаз
(`LKOH ROSN SIBN TATN GAZP NVTK`), металлурги (`GMKN NLMK MAGN CHMF`) и `SBER`.
Сканер сам выберет лучшую пару. Свою корзину можно задать через `--tickers`.

## Файлы

| файл | назначение |

| `st6_core.py` | ядро: статистика, отбор пар, FSM, сайзинг, P&L (без сети) |

## Установка

```bash
pip install "tinkoff-investments>=0.2" numpy statsmodels
export INVEST_TOKEN=t.xxxxxxxx        # токен T-Bank Invest с доступом к песочнице
```

`statsmodels` опционален: с ним коинтеграция считается настоящим ADF-тестом,
без него — встроенным Dickey-Fuller (lag 0) по критическим значениям DF.

## Запуск

```bash
python st6_bot.py scan                 # показать лучшие пары корзины
python st6_bot.py backtest             # бэктест лучшей пары на истории T-Bank
python st6_bot.py live --dry           # торговый цикл БЕЗ отправки ордеров
python st6_bot.py live                 # торговый цикл в песочнице (sandbox-ордера)
python st6_bot.py live --once          # один проход цикла и выход
```

Полезные флаги: `--interval 1h` (1m/5m/10m/15m/1h/4h/1d), `--days 200`,
`--tickers LKOH ROSN SIBN`, `--entry 2.0`, `--stop 3.5`, `--risk 0.02`.

> ⚠ T-Bank отдаёт минутные/10-минутные свечи ограниченным окном (~7 дней).
> Для глубокой истории берите `--interval 1h` или `1d`.

## Параметры (`st6_core.Params`)

| параметр | по умолч. | смысл |
| -------- | --------- | ----- |
| `beta_window` | 240 | окно OLS-оценки β и средней спреда |
| `z_window` | 240 | окно z-score |
| `corr_window` | 120 | окно скользящей корреляции |
| `z_entry / z_exit / z_stop` | 2.0 / 0.3 / 3.5 | пороги входа / выхода / стопа |
| `corr_enter / corr_break` | 0.80 / 0.55 | корреляционный гейт входа / аварийного выхода |
| `select_min_corr` | 0.80 | мин. корреляция при отборе пары |
| `select_max_pvalue` | 0.10 | макс. ADF p-value (коинтеграция) |
| `select_max_halflife` | 240 | макс. half-life возврата (бары) |
| `risk_fraction` | 0.02 | доля equity на нотионал каждой ноги |
| `fee_rate / slippage_rate` | 0.0006 / 0.0005 | издержки на оборот ноги |

## Тесты

```bash
python test_st6.py        # 17/17 PASS
```

Покрывают: восстановление β по OLS, корреляцию связанных/независимых рядов,
z-score, half-life, знак P&L по обеим сторонам, влияние издержек, доллар-нейтральный
сайзинг, все ветки FSM (вход по z, блокировка гейтом, выход по corr-break и по цели),
отбор коинтегрированной пары и детерминизм бэктеста (no look-ahead).

## Честно про риски

Высокий win-rate бэктеста объясняется силой коинтеграции на спокойном периоде.
Главные риски: **слом коинтеграции** (страхуется corr-гейтом, но не полностью),
**зависшая позиция** при медленном возврате спреда, и **издержки** на частых
ребалансах. Метрики на синтетике ничего не говорят о реальной доходности — это
проверка, что логика считается верно. Боевой контур не подключается: только
SandboxService.
